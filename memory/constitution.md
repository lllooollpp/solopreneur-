# Nanobot 项目宪章

## 项目名称
🐈 Nanobot - 个人 AI 助手

## 版本
1.1.0

## 治理日期
- 原始采用日期 (Ratification Date): 2026-02-05
- 最后修订日期 (Last Amended Date): 2026-02-05

## 核心原则

### 1. 透明记忆 (Transparent Memory)
**MUST**: 所有 Agent 的记忆数据必须以人类可读的 Markdown 格式存储在工作空间中。
**理由**: 确保用户对 AI 记住的内容拥有完全的知情权和物理控制权，降低系统黑盒化风险。

### 2. 中文化标准 (Chinese Localization)
**MUST**: 项目的所有 CLI 交互、日志输出、核心代码注释以及工作空间模板必须原生支持中文。
**理由**: 本项目定位于服务中文开发者社区，消除语言隔阂是提升协作效率的关键。

### 3. 异步总线通信 (Async Bus Communication)
**MUST**: 通信通道（Telegram/WhatsApp 等）与 Agent 逻辑之间必须通过异步消息总线 (`Message Bus`) 进行解耦。
**理由**: 保证系统在高并发或网络波动下的稳定性，并支持轻松水平扩展新的通信渠道。

### 4. 受控子 Agent (Controlled Subagents)
**MUST**: 当执行 A2A (Agent-to-Agent) 协作时，子 Agent 必须运行在能力受限（无用户交互权限）的沙箱中。
**理由**: 防止自动化任务失控或造成敏感信息误分发，确保主 Agent 的审查权。

### 5. 极简配置 (Minimalist Configuration)
**MUST**: 系统配置优先依赖于工作空间内的 `.md` 和 `.json` 文件，禁止引入臃肿的外部配置服务。
**理由**: 保持项目的轻量化和“开箱即用”特性，降低部署门槛。

## 治理
- **修订程序**: 任何实质性的架构变动或原则增删必须同步修订本宪章并递增版本号。
- **合规审查**: 在执行 `speckit.plan` 时，必须核对任务是否违反上述核心原则。
- **版本控制政策**: 遵循语义化版本规则（SemVer）。主版本变更需经过核心开发者一致同意。
