# 记忆系统 (Memory System)

solopreneur 采用了一种基于 **Markdown 文件** 的轻量级记忆管理系统，而非传统的数据库（如 SQLite）。这种设计使得记忆内容对用户完全透明，且易于手动编辑和备份。

## 1. 核心存储结构

记忆文件存储在工作目录的 `memory/` 文件夹中：

-   `memory/MEMORY.md`: **长期记忆**。存储关于用户及其偏好的核心、持久信息。
-   `memory/YYYY-MM-DD.md`: **每日笔记**。按日期记录 Agent 的日常活动、重要事件和反思。

## 2. 运作机制

### 2.1 记忆注入 (Injecting Context)
在 Agent 的每个对话循环中，系统会自动整合记忆：
1.  读取 `MEMORY.md` 中的长期记忆。
2.  读取 `today.md` 中的当日笔记。
3.  将这些内容格式化为 Markdown 文本段落。
4.  将其作为对话上下文的一部分发送给大语言模型 (LLM)。

### 2.2 记忆更新 (Updating Memories)
Agent 具备主动管理记忆的能力：
-   **追加记录**: Agent 可以将新发现的信息追加到当天的日志中。
-   **持久化反思**: 经过一段时间后，Agent 可以总结当天的活动并更新到长期记忆中，以保持上下文的精简和高效。

## 3. 设计优势

1.  **用户中心化**: 用户可以使用任何 Markdown 编辑器直接修改 Agent 的记忆。
2.  **版本控制友好**: 所有的记忆都是纯文本，可以轻松通过 Git 进行版本追踪。
3.  **零依赖**: 无需安装或配置数据库服务，系统启动即用。
4.  **隐私可见**: 用户可以清楚地看到 Agent 记住了什么，没有黑盒操作。

---

*注意：虽然目前基于文件，但系统架构允许未来扩展到向量数据库 (RAG) 以处理海量历史信息。*
